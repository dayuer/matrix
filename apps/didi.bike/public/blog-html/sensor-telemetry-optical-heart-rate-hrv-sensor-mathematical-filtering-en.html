Understanding Optical Heart Rate HRV Sensor through Mathematical Filtering

1. Introduction and State Estimation Framework

This paper examines the mathematical modeling and signal processing structures utilized in high-frequency mobile biotelemetry. Specifically, the dynamic calibration of an Optical Heart Rate HRV Sensor presents non-trivial challenges regarding signal propagation and noise extraction in embedded system designs. By deploying Mathematical Filtering techniques, it is possible to accurately estimate the rider's pitch, roll, and linear acceleration from noisy accelerometric and gyroscopic data streams.

For telemetry architectures designed for aerodynamic estimation (CdA), the minimization of sensor drift and barometric latency is a requirement. Steady-state stability under variable grade conditions must be maintained to ensure the integrity of the downstream sports science calculations.

2. Mathematical Modeling and State Optimization

To address sensor noise and integration drift, the telemetry processor employs a discrete state-space estimation model:

Where:

  • $x_k$ represents the estimated state vector (e.g., rider attitude or elevation), estimated recursively using a Kalman filter.
  • $f_{\text{nyquist}}$ is the minimum sampling frequency required to capture high-frequency pedal vibrations without aliasing, according to the Nyquist-Shannon theorem.
  • $V_{\text{comp}}$ represents the temperature-compensated sensor voltage output, correcting drift using a polynomial calibration coefficient $\alpha$.
  • $q_k$ represents the quaternion vector used to calculate Euler angles without gimbal lock.

The recursion parameters within the Kalman filter ($x_k$) govern the response behavior of the system. Furthermore, temperature variations are compensated mathematically by adjusting $V_{\text{comp}}$ via the coefficient $\alpha$, while the quaternion product $q_k$ prevents singular configurations in the orientation matrix.

3. Signal Processing Implementation and Results

The data filtering and processing pipeline consists of three specific components:

  1. 6-Axis Sensor Fusion complementary model: The fusion of triaxial accelerometers and gyroscopes compensates for high-frequency gyro drift and low-frequency accelerometric noise, stabilizing the Euler angle approximations.
  2. Gravity Subtraction Vector derivation: Dynamic subtraction of the gravity vector from raw acceleration measurements is performed to isolate the true linear acceleration forces generated by the cyclist.
  3. Low-Power Firmware Compression scheme: Lossless compression via run-length encoding reduces ANT+/BLE transmission bandwidth, optimizing power consumption while maintaining data integrity at high sampling rates.
